# Local MCP Proxy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local-only MCP proxy so Claude Code, Codex, and Hermes can share Mobbin and Refero without opening competing upstream Mobbin sessions.

**Architecture:** The proxy exposes two local Streamable HTTP MCP endpoints and keeps one upstream MCP client per provider. Tool calls pass through a per-provider FIFO queue, an in-flight de-duplication map, and a file-backed cache keyed by provider, tool name, and normalized arguments. Mobbin defaults to a single `mcp-remote` upstream child process for OAuth; Refero defaults to Streamable HTTP with a bearer token env var.

**Tech Stack:** Node.js 24, TypeScript, `@modelcontextprotocol/sdk`, Express, Vitest.
